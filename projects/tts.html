Current experience

The feedback we received from our users was they could not perform various actions while using TTS. In addition, we think the exsiting visual was outdated and dark.



"I wanted to add a bookmark for a page in my favorite chapter while using TTS, but I can not do that, I want to listen while reading" (feedback from Xinyu)

Insight1

Lack of flexibility to peforming actions

We can see users quit and restart constantly from the data. Users need quit listening first and then peform actions based on current user flow.

Insight2

Lack of visibility

Current visual design has issue with contrast for accessibility.






What's users real listening needs?

I collaborated with data analyst to understand what's the current existing data for TTS. I found 70% users tap middle screen during listening and can clearly see the data of adjust the setttings decreased after the first time when users setting.


Only 6.6% users adjust timer/voice/speed and rarely use after they set up frist time


Insight

A.Need deprioritize setting features and rethink user flow when tap the middle sreen

B.Need enhance the contrast for accessibility


90% users click those settings for first time and I see the data of adjust the setttings deeply decreased after the first time compared to second time and third time. We can concluded that the main reason of low click rate did not due to lack of contrast.


USER FEEDBACK & INTERNAL TESTING

Then I collected some user feedback from customer service department and discovered TTS was rated as #1 among the negative experiences. Thus we conducted an internal testing to understand user's needs when using TTS.

internal user interview

Insights

  • Users prefer multitask ability instead of just performing reading actions
  • Users need quick actions to control listening
  • Users can not see entry point clearly since background was dark and outdated


Reframe the design goal

Improve the TTS visual and flow to allow users to explore with other pages while listening





Finding a Frictionless listening flow to fit users' mental modal

solution 1

Add a new TTS player on the navigation bar which is available for all pages


Review findings about Solution 1

  • Users may become confused about two audio entry points as the navigation bar already has entry point for audio books
  • The navigation items might get overlapped in certain pages
  • Both the TTS and audiobook entry points have animation and users may feel overwhelmed
  • Users need additional steps to control audios
  • Users can perform other tasks while listening to the audio

solution 2

Use a floating audio player component

Review findings about Solution 2      Move Forward

  • Users can perform other tasks while listening to the audio
  • Users can control the audio in one place for both audio books and TTS
  • Users can quickly play/pause/quit the audio in place
  • Users seamlessly switch between the control pages of TTS and audio books
  • We need to build a new player page for TTS as it is not compatible with the existing one

Finalized Design

Frictionless listening flow

Users now can multitasking without quitting listening. These feature will allow users explore and discover content.



Enhance and move TTS entry point from top to bottom

Listeners now reach and start TTS easily. We also redesign to a light grey setting background color to enhance visibility.



Change control setting into entire page

Finding a flow fits users' mental modal and provide more books to discover.


Reorganize the options in voice change section

Regroup voice options to enhance efficiency.



Reorganize the options in timer section

Allow users set timer directly instead of scrolling.

Impact

A/B Testing

Controlled Experiments -Stage 1

In order to text our design assumptions, we created controlled experiments. We expected that the winner would acquire the highest listenting conversion rate (the number of people who reached this page/the number of people who tapped the "listen" button).

we decided to release design step by step

We want to know exactly which design decision leads to the changes in metrics Instead of launching whole design at one time.


After two weeks observation, the data showed: Significant increase in the listenting conversion ratio in the experimental group and no negative effects on other metrics, so we fully launched the design of the experimental group.


Controlled Experiments -Stage 2

This time we proceeded further. The previous "experimental group" became the "control group", and we created other new experimental groups

Compared with the control group, we saw:

  1. +3% TTS usage rate and time spent per user
  2. 97% user used TTS outside the reader
  3. Retention rate of whole listening users has been slightly increased

We did not see any negative results, all the metrics were positive, so we fully launched our new TTS flow and unitized audio play component
