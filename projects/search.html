design process

challenge 1

How might we provide additional blurry searching?

"I only remembered the character name, but could not find that books" (feedback from our user Qing Li)

What are users usually search for?

28.4% key words matched with character names.


Based on the user feedback above, we knew that users desire more searching functionalities. And thus, we did the seraching analysis and found that 28.4% key words matched with character names. Therefore, we decided to support searching by characters as the first priority to help users find the right content quickly as well as help us to stand out from other competitors.

Ideation

We know that we need have character search feature, but what specific about character will work? I collabrorated with other designers, pm and engineers to brainstorming some board ideals, we choose character label and character name highlighted to deep dive into next step as they are really targeting in searching process. We don't moveforad with character name highlighted as book photos attract too much users’ attention making this design less effiective. Character label has mutiple advantages.

Solution

Enhance blurry Search via Seaching by Character label

Character Label

Add a label to the suggested words.

So what is a Character Label? It is a label indicating there will be at least one book if users search by the character with this key word. The label is iconified to facilitate users to locate the right search option. And with color coding, we can easily extend it to have more options to satisfy new searching requirement in future. Tapping on it will take users to the character search result page.

Provide searching by character option in the top of the typeahead dropdown, tapping on it will open a new page to display the searching result based on character.

Design review

Project team review

Customer report

  • The tag with solid background is more discoverable
  • Extra information in result page can further help users find the right books
  • Easily extend the searching capabilities by applying different color coded labels
  • Low visual and informational efficiency when displaying multiple searching results
  • The changes to the order of suggested words require more engineering refactoring

Conclusion: Considering the drawbacks of Character Label are something improvable while the ones of Highlighted Character Name are not, so I decided to move forward with Character Label.

How many key suggested words should be displayed?

  • Remained the same order when displaying the regular suggested words
  • Reduced the number of suggested words from 5 to 3 in order to bubble up the Character Label in the dropdown based on the data we found that users generally interacted with top 3 words in first section
  • Extended the label to include Content Tagging and Book Category

How to effectively display the information?

The main goal for result page is helping users QUICKLY locate the RIGHT content. And the previous layout of book information did not follow the importance of information. Therefore, I reorganized the information (e.g. moved the rating to the top right), see below.

Since we supported searching by Character Label, how do we associate the search results with it? Based on this, I came up with three different options:

  • option1: Put Character Label for all of the books.
  • option2: Put Character Label next to the book name for each book.
  • option3: Put character name within other assistant data, and highlight it.

Option 1 has less strong connection between characters and books while Option 2 does not visually align very well as the length of book names could vary often times. Option 3 does not have these drawbacks, most imporantly, with highlighted text, users tend to read other assitant information which helps them better understand and make the decision. Thus we chose the Option 3 as the final design.

challenge 2

How might we reduce noise in search recommendation page?

"Why does it recommend so many books that I am not interested in? I only wanted to read the one I am looking for" (feedback from our user Qing Li)

Problem insight

Recommended Search for You (in pre-seraraching page) had only 2% CTR.

Hot Search (with images) had a much higher CTR but low exposure

We used to borrow the ideas of social media apps, and built many feed recommendations across Home page and Search page for the purpose of user engagement. However, we have seen the product cannibalization between these two pages.

Hypothesis

  1. Avoid product cannibalization
  2. Users might be more attracted by images than text
  3. Users tend to be more interested with the hot topics

Solution

Consolidate Search Recommendation

I suggested to remove the “Recommend for You” in pre-searching page to avoid cannibilization with Feeds, which also freed spaces for other high signal modules. Also considering the Hot Search was overlapped by keyboard, I suggested to consolidate it with Catogory list.

feedback

After the initial public launch, we noticed the exposure volume remained low due to lack of awareness of the test. Thus we added a few more upsells to introduce the new functionalities for test users.

  1. Upsell the Character name seraching in the placeholder of search box
  2. Upsell the Character name searching next to the searching entry point

We also found that usually the first matched book is the one users are looking for, and usually that book is on Hot Search leaderboard. And thus, we proposed two more ideas:

  1. Display the book name in the search dropdown if the character name is 100% matched, tapping on the book name will take users directly to book detail page;
  2. Add CTA to Hot Search leaderboard under the book if it is on the leaderboard;
